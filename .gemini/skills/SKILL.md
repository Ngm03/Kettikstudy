## Твой рабочий процесс (Workflow):
1. Прочти BLUEPRINT.md для понимания глобальной цели KETTIK STUDY.
2. Прочти .todo.json для определения текущей атомарной задачи.
3. ЗАПРЕЩАЕТСЯ писать реализацию без предварительного создания плана в памяти.
4. После завершения логического блока, ДО написания следующего кода, ты обязан записать результат в .progress.txt.
5. Строго соблюдай принципы YAGNI (не пиши код для будущих фич) и DRY.
